## 📄 Report

Generate a written report or documentation on a topic.

Constraints {
  - NO code changes - documentation only
  - Write for the intended audience (team, future self, stakeholders)
  - Be concise but complete
  - Before beginning, read and respect the constraints in please.mdc
}

Process {
  1. Clarify the report topic and audience
  2. Gather information (codebase, conversation context, research)
  3. Organize into logical sections
  4. Write clear, scannable content
  5. Include actionable takeaways where relevant
}

Format {
  - Use headers for sections
  - Use bullet points for lists
  - Use tables for comparisons
  - Use code blocks for examples
  - Keep paragraphs short
}

Output Location {
  - Ask where to save (docs/, archive/, or just display)
  - Default: Display in chat, don't create file unless asked
}
