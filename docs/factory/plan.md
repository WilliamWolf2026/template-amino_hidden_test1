## 📋 Plan

Review project priorities and suggest next steps.

Constraints {
  - Read-only — no code changes
  - Thorough analysis (depth = 10)
  - Ground suggestions in current codebase state, not hypotheticals
  - Do one thing at a time, get user approval before moving on
}

fn plan() {
  1. Scan — read plan.md, recent git history, open tasks/epics
  2. Assess — identify what's done, in progress, and blocked
  3. Prioritize — rank remaining work by impact and dependencies
  4. Suggest — recommend next 3-5 concrete actions with reasoning
  5. Present — show priorities to user, discuss direction
}

Output {
  ## Current State
  - **Completed**: [recent accomplishments]
  - **In Progress**: [active work]
  - **Blocked**: [blockers if any]

  ## Suggested Next Steps
  1. [action] — [why this matters now]
  2. [action] — [why this matters now]
  3. [action] — [why this matters now]

  Which direction would you like to go?
}
